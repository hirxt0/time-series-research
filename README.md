## Установка и запуск

### Зависимости

```txt
python <= 3.11
fastapi
uvicorn[standard]
python-multipart
pandas
numpy
scikit-learn
lightgbm
timesfm==1.3.0
```

Установить все зависимости:

```bash
pip install fastapi uvicorn[standard] python-multipart pandas numpy scikit-learn lightgbm timesfm==1.3.0
```

### Запуск

```bash
cd GeoMagAnalyst
uvicorn main:app --reload --port 8080
```

Открыть в браузере: [http://localhost:8080](http://localhost:8080)

##  Запуск через Docker

```bash
# Собрать образ
docker build -t geomag-analyst .

# Запустить контейнер
docker run -p 8080:8080 geomag-analyst
```

###  Проблема
Реальные данные магнитных станций содержат **пропуски** (сбои оборудования, перебои питания) и **инструментальные артефакты** (импульсные выбросы без физической природы). Ручная обработка трудоёмка и не масштабируется на объёмы современных многолетних архивов (миллионы точек).

###  Данные
- **Источник:** записи магнитометрической станции ARS (2023–2025 гг.).
- **Объём:** ~10.5 млн точек в год (исходный шаг 3 сек).
- **Предобработка:** ресэмплинг до 1-минутного разрешения (усреднение по 20 точкам), что сокращает объём в 20 раз без потери информативности для задачи импутации.

##  Технический Pipeline

### 1. Детекция аномалий (Isolation Forest)
- **Задача:** отделить инструментальные артефакты от реальных геомагнитных бурь в полностью *unsupervised* режиме (без разметки).
- **Решение:** алгоритм на 3 признаках:
  - `value` (исходное показание)
  - `delta` (модуль разности соседних точек, мгновенная производная)
  - `v_long` (скользящее стандартное отклонение за 1 час, контекст бури)
- **Логика:** буря имеет высокую волатильность вокруг скачка, артефакт — нет.

### 2. Feature Engineering (19 признаков)
Все *rolling*-статистики вычислены со сдвигом `shift(1)` для предотвращения утечки данных:
- **Временные и циклические:** час, день недели, номер недели, месяц, `sin`/`cos` кодирование часа и дня года.
- **Лаговые:** скользящие средние за 30 мин, 1 ч, 3 ч, 6 ч, EMA.
- **Производные:** наклоны (`slope`), разности (`diff`) и индикатор бурь (`rolling_std_24h`).

### 3. Импутация (заполнение пропусков)
Сравнение подходов на синтетических пропусках (хронологическое разделение 85%/15%).

##  Сравнение моделей (Метрика MAE)

| Модель | MAE (нТл) | RMSE (нТл) | Статус / Комментарий |
| :--- | :--- | :--- | :--- |
|  LightGBM |317 | 375 | Лидер (Production).** Лучшее качество за счёт обучения на целевом распределении и богатого признакового пространства. |
|  **CatBoost** | 420 | 658 | Хорош на коротких пропусках, но деградирует на длинных (> 6 ч) из-за исчезновения лаговых признаков. |
|  TimesFM 2.5 | 1 476 | 1 711 | Foundation-модель Google (200M параметров, zero-shot). Двунаправленный проход в 2 раза лучше прямого. |
|  *Baseline* |2 593 | 3 251 | Линейная интерполяция. |
|  **N-HiTS** | 3 650 | - | Экспериментальная нейросеть. Хуже базовой интерполяции без явного feature engineering. |

>  **Ключевой вывод:** Градиентный бустинг (LightGBM) с инженерными признаками **в 4.6 раза точнее**, чем топовая foundation-модель (TimesFM) в zero-shot режиме.

##  Веб-приложение

- **Стек:** Python 3.11, FastAPI, uvicorn, Chart.js, Docker.
- **Функционал:** 
  1. Загрузка CSV-файла с сырыми данными.
  2. Автоматический ресэмплинг и feature engineering.
  3. Детекция аномалий (Isolation Forest) с визуальной разметкой.
  4. Выделение периодов геомагнитных бурь.
  5. Заполнение пропусков выбранной моделью (LightGBM / CatBoost / TimesFM).
  6. Интерактивный график: исходный ряд, аномалии, пропуски и восстановленные сегменты.
